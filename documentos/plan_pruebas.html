1. Objetivo del plan de pruebas

El presente plan define la estrategia, alcance, recursos y criterios para verificar que el prototipo de alta fidelidad del Sistema de Gestión Aduanera (SGA) cumple con los estándares de calidad establecidos en la norma ISO/IEC 25000 (SQuaRE) y los atributos de calidad definidos en ISO/IEC 9126, satisfaciendo los requisitos funcionales y no funcionales levantados a partir del caso de estudio del paso fronterizo Los Libertadores.

2. Alcance

Módulos incluidos en las pruebas:

  • Módulo de autenticación (Login)
  • Dashboard y panel de control
  • Registro de menores de edad
  • Control de vehículos motorizados
  • Revisión SAG (alimentos, mascotas) y PDI
  • Integración con sistemas de países limítrofes
  • Estadísticas e informes
  • Generación de reportes (PDF/Excel)
  • Gestión de usuarios y perfiles

Fuera del alcance: Pruebas de rendimiento bajo carga real, integración con bases de datos productivas (el prototipo es de alta fidelidad, sin backend real).

3. Normativa aplicada

NormaDescripciónAplicación en este proyecto
ISO/IEC 25010
(parte de ISO 25000)		 Define el modelo de calidad del producto software con 8 características principales. Base para definir qué características de calidad se prueban (usabilidad, funcionalidad, seguridad, etc.)
ISO/IEC 25040
(parte de ISO 25000)		 Proceso de evaluación de calidad del software. Guía el proceso: planificación → diseño de casos → ejecución → registro de resultados.
ISO/IEC 9126 Modelo de calidad predecesor al ISO 25010. Define 6 características de calidad. Referencia para los atributos de funcionalidad, confiabilidad, usabilidad, eficiencia, mantenibilidad y portabilidad.
